# Continuous Price

D10 builds `research_continuous_price_daily` for signal objects such as `CF.C1`.

## Inputs

- `core_quote_daily` rows from normalized quote fixtures.
- `core_chain_map_daily` rows generated by D8.

Research code must not parse raw exchange payloads. D10 consumes only normalized
core rows.

## Adjustment Rule

The MVP uses additive back adjustment:

- On unchanged mapping days, `adjusted_price = raw_price + cumulative_adjustment`.
- On roll days, D10 compares the new mapped contract and previous contract on
  the same `trade_date`.
- `roll_gap = new_contract_price - previous_contract_price`.
- `adjustment = -roll_gap`.
- `cumulative_adjustment` is applied to the roll day and all later rows.

This keeps the continuous series smooth for research factors while preserving the
real mapped contract in every row.

## Trace Fields

- `mapped_contract`
- `raw_price`
- `adjusted_price`
- `adjustment`
- `cumulative_adjustment`
- `is_roll`
- `roll_from_contract`
- `roll_to_contract`
- `roll_gap`
- `chain_switch_reason`
- `input_snapshot_ids`

Continuous prices are signal objects only. Orders, fills, costs, and positions
must use D9 `core_trade_mapping_daily`.

## CLI

```bash
$env:PYTHONPATH="src"; py -3.12 -m cotton_factor.cli.main core build-continuous-price --product CF --start 2024-01-09 --end 2024-01-12 --quote-fixture tests/fixtures/core_quote_daily_cf_chain_sample.csv --ltd-buffer-days 2
```
